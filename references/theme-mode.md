# Theme Mode

Use this mode to deepen one capability across multiple experiences.

## Good Theme Examples

- cross-functional collaboration
- ownership
- architecture decisions
- conflict handling
- performance tuning
- mentoring

## Goal

Build an evidence chain that proves the user has repeated strength in the theme, not just one isolated anecdote.

## Workflow

1. define the theme clearly
2. identify which files contain evidence for it
3. ask one question at a time to fill missing pieces
4. compare examples and remove overlap
5. distill the strongest two or three stories

## Question Strategy

Prefer questions that force differentiation:

- which project best proves this theme
- what made your action distinct from the team's work
- where did this theme create measurable impact
- what trade-off or conflict made this theme credible

## Output

Build a theme note in `deep-interview/themes/<theme-slug>.md` containing:

- theme definition
- strongest supporting experiences
- confirmed facts per experience
- best interview framing
- weak spots still needing evidence

Use [theme-template.md](theme-template.md) for the file structure.

## Stop Condition

Stop when the user has enough evidence to answer the theme clearly with more than one example and without repeating the same story twice.

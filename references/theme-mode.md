# Theme Mode

Use this mode to deepen one capability across multiple experiences.

## Good Theme Examples

- cross-functional collaboration
- ownership
- architecture decisions
- conflict handling
- performance tuning
- mentoring

## Goal

Build an evidence chain that proves repeated strength in the theme, not just one isolated anecdote.

## Workflow

1. Define the theme clearly
2. Identify which files in `drafts/` contain evidence for it
3. Ask one question at a time to fill missing pieces
4. Compare examples and remove overlap
5. Distill the strongest two or three stories

## Question Strategy

Use self-questioning to uncover evidence:

- Which project best proves this capability?
- What made my action distinct from the team's work?
- Where did this capability create measurable impact?
- What trade-off or conflict makes this capability credible?

## Output

Build a theme note in `deep-interview/themes/<theme-slug>.md` containing:

- theme definition
- strongest supporting experiences
- confirmed facts per experience
- best interview framing
- weak spots still needing evidence

Use [theme-template.md](theme-template.md) for the file structure.

## Stop Condition

Stop when there is enough evidence to answer the theme clearly with more than one example and without repeating the same story twice.
